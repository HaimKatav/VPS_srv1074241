# Wiki Information Architecture

> This is the **canonical routing page** for the TraderB memory-wiki vault. When ingesting or creating any page, consult this page first. Route by **type**, not by **topic**.

## The five top-level folders (memory-wiki native)

- `sources/` — raw evidence. Immutable in spirit. Ingested from original docs, agent history, bridge imports from memory-core. Pages never move out; they're referenced.
- `entities/` — named things. Servers, services, sub-projects, people, accounts. The nouns.
- `concepts/` — ideas, principles, rules, do/don't. The how we think. Including architecture do/don't, coding conventions, UX principles.
- `syntheses/` — aggregated or derived knowledge. Specs, milestones, postmortems, journal. What we built, decided, learned.
- `reports/` — **plugin-owned**. Auto-generated by memory-wiki on compile/lint. Do NOT hand-edit.

## Domain sub-structure

| Knowledge kind | Folder |
|---|---|
| Architecture principles, parity rules, abstraction patterns | `concepts/architecture/` |
| Coding conventions, commit format, Python style, module layout | `concepts/coding/` |
| UX principles (hierarchy, trader-readable rules) | `concepts/ux/` |
| Meta / how this vault works | `concepts/meta/` (this folder) |
| Servers, endpoints, ports, credential locations | `entities/infra/` |
| Services (dashboard, observatory, vikunja, openclaw-ui, wiki) | `entities/services/` |
| Sub-projects as named things (compliance-agent, pro-trader-mcp, observatory) | `entities/projects/` |
| Specific UI / dashboard specs (e.g. P&L panel v3) | `syntheses/ux/` |
| Sub-project working specs / roadmaps | `syntheses/projects/<name>/` |
| Completed milestone close-outs (M1, M2...) | `syntheses/milestones/` |
| Postmortems, retros, lessons learned | `syntheses/postmortems/` |
| Agent daily/weekly reflections | `syntheses/journal/<YYYY-MM-DD>.md` |
| Auto-compiled from OpenClaw memory-core | `sources/bridge/` |

## Tiebreaker rule (when unsure)

- If it's a **rule** → `concepts/`
- If it's a **thing** → `entities/`
- If it's **derived** (aggregation, spec, narrative) → `syntheses/`
- If it's **raw evidence** → `sources/`
- If it's **auto-generated** → `reports/` (plugin-owned)

## Provenance discipline (re-stated from governance)

Every page's frontmatter carries exactly one provenance marker: `extracted`, `inferred`, `ambiguous`, or — for trading-relevant pages — `approved-by-haim: true`.

Agents may read lower-confidence pages for context. Agents must NEVER cite an `inferred` or `ambiguous` page as authority for code that trades real money.

## Who writes where

- **Librarian (Ari)** — only persona with unprompted write access to the vault.
- **Other personas** — contribute via handoff files (`openclaw/handoffs/`). Ari ingests, curates, promotes.
- **System cron** — writes via deterministic ops: `openclaw wiki compile`, `openclaw wiki lint`, nightly git snapshot + push.
- **Plugin** — writes to `reports/` and `.openclaw-wiki/` (cache, state, logs).
